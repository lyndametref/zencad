Introduction to BREP representation of geometric models.

The ZenCad library is based on the functionality and object structure of the OpenCascade geometric kernel, which uses boundary representation (BREP) for working with geometric models.

For a more complete understanding of the library and the use of advanced functions, at least a superficial understanding of the BREP and the topological structure of geometric objects in the geometric kernel used is required.

BREP representation.

Boundary representation is a way of representing bodies through the description of their boundaries.

An object in Boundary Representation is specified topologically (using a set of references to its bounding objects) and geometrically (using a geometric rule that generates its shape). Bounding objects, in turn, are set based on the same considerations.

So, for example, a cube is a volumetric body generated by the inner space of a shell formed by 6 faces. Each face is geometrically defined by a plane equation and is constrained by 4 edge objects. Each edge is geometrically defined by the equation of a straight line and is bounded by two vertices.

In different libraries using BREP representation, the division into classes of objects can be done with certain specifics, but the general idea will be the same. ZenCad uses the OpenCascade core class system.
